# Algo Trading Academy — Правила системы

Ты работаешь внутри учебного проекта по алготрейдингу на базе Freqtrade.
Задача: помочь ученику разрабатывать, тестировать и запускать торговые стратегии
внутри Docker-контейнера. Всё должно быть безопасно — ученик может сломать
только свои стратегии, но не инфраструктуру.

## Среда

- **Freqtrade** 2026.3 в Docker-контейнере (образ `freqtradeorg/freqtrade:2026.3`)
- **Entry point:** `docker compose up -d` (единственный способ запуска)
- **FreqUI:** http://localhost:8080 (логин: `freqtrader` / `freqtrader`)
- **Режим:** dry-run (виртуальные деньги) по умолчанию
- **Биржа:** Binance (spot)
- **Данные:** parquet-формат, хранятся в `user_data/data/binance/`

## Защищённые файлы

Эти файлы нельзя изменять, потому что они составляют стабильную инфраструктуру,
общую для всех учеников. Поломка любого из них = ученик не может работать.

| Файл | Почему нельзя трогать |
|---|---|
| `docker-compose.yml` | Меняет порты, volumes или healthcheck — контейнер может не подняться |
| `user_data/config/config.json` | Базовый конфиг dry-run. Если сломать — бот не стартует |
| `.env` | Содержит секреты (API-ключи). Не показывать, не коммитить. Для dry-run `.env` не нужен — не создавай и не требуй его |
| `.gitignore` | Контролирует что попадает в git |

### Как менять конфигурацию

Freqtrade поддерживает мульти-конфиг: несколько `--config` файлов мержатся слева направо.
Базовый `config.json` остаётся нетронутым. Для изменений:

1. Создай `user_data/config/config.override.json` с нужными параметрами
2. Явно передай оба конфига при запуске команды:
   ```bash
   docker compose run --rm freqtrade backtesting \
     --config /freqtrade/user_data/config/config.json \
     --config /freqtrade/user_data/config/config.override.json \
     --strategy ИмяСтратегии
   ```

Override НЕ подхватывается автоматически — всегда указывай `--config` явно.

Пример override для увеличения кошелька:
```json
{
    "dry_run_wallet": 5000,
    "max_open_trades": 5
}
```

## Защищённая структура

Не создавай и не удаляй папки внутри `user_data/`. Структура соответствует
стандарту Freqtrade и CLI ожидает конкретные пути:

```
user_data/
├── config/          # конфиги (config.json + override)
├── strategies/      # стратегии (.py файлы)
├── data/binance/    # исторические данные (parquet)
├── backtest_results/# результаты бэктестов
├── logs/            # логи бота
└── tradesv3.sqlite  # БД трейдов (автосоздаётся)
```

Создавать файлы внутри существующих папок — можно (новые стратегии, конфиги и т.д.).

## Стратегии — правила работы

- **Перед перезаписью** существующей стратегии — сделай бэкап:
  `cp Strategy.py Strategy.py.bak`
  (мелкие правки внутри файла бэкапа не требуют — git отследит)
- **Новые стратегии** создавай в `user_data/strategies/` с уникальным именем класса
- Имя класса в файле ДОЛЖНО совпадать с тем что передаётся в `--strategy`
- Добавляй `can_short = False` если стратегия только для spot

## Команды Freqtrade

Все команды запускать ТОЛЬКО через Docker:

```bash
# Бэктест
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/user_data/config/config.json \
  --strategy ИмяСтратегии \
  --timerange 20260101-20260401

# Скачать данные
docker compose run --rm freqtrade download-data \
  --pairs BTC/USDT ETH/USDT \
  --exchange binance \
  --days 90 -t 1h \
  --config /freqtrade/user_data/config/config.json

# Список доступных пар
docker compose run --rm freqtrade list-pairs \
  --config /freqtrade/user_data/config/config.json \
  --exchange binance

# Логи работающего бота
docker compose logs -f --tail 50
```

Не устанавливай пакеты вне контейнера. Не запускай `pip install` на хосте.
Если нужна библиотека — она уже есть в образе (ta-lib, technical, pandas, numpy, ccxt).

## Безопасность

- **Live-торговля:** не запускай без ДВОЙНОГО подтверждения пользователя.
  Переспроси: "Вы точно хотите запустить реальную торговлю? Это будет использовать
  настоящие деньги."
- **API-ключи:** никогда не показывай содержимое `.env` в чате
- **Защищённые файлы и live:** перед изменением любого защищённого файла
  (см. таблицу выше) или запуском live-торговли — опиши план и получи подтверждение

## Проверка ОС

Поддерживаемые сценарии: **macOS** и **Linux** (docker compose нативно).
Windows поддерживается только через WSL2 — предупреди о возможных проблемах с путями.

Не предлагай команды и пути, специфичные для неподдерживаемых ОС.
Если не уверен в ОС пользователя — спроси перед выполнением системных команд.
