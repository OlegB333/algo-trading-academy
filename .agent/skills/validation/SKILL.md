# Валидация стратегии

## Зачем

Бэктест на одной паре за один период — это только начало. Хороший результат
может быть случайностью или подгонкой. Валидация отвечает на вопрос:
"будет ли стратегия работать в реальности?"

## Когда валидировать

- После ручной оптимизации (изменил параметры → проверь)
- **Обязательно после hyperopt** — автоматическая оптимизация особенно склонна
  к подгонке, валидация на отложенных данных критична (см. skill `hyperopt`)
- Перед переходом на dry-run / live

## Чек-лист валидации

### 1. Out-of-sample тест

Раздели данные: 70% для разработки, 30% для проверки.

```bash
# Разработка (тренировка)
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/user_data/config/config.json \
  --strategy MyStrategy \
  --timerange 20250601-20260101

# Проверка (out-of-sample)
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/user_data/config/config.json \
  --strategy MyStrategy \
  --timerange 20260101-20260401
```

Результат на проверочном периоде не должен быть драматически хуже тренировочного.
Небольшая просадка — нормально, полный развал — проблема.

### 2. Тест на другой паре (опционально)

```bash
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/user_data/config/config.json \
  --strategy MyStrategy \
  --pairs ETH/USDT \
  --timerange 20260105-20260405
```

Некоторые стратегии заточены под конкретный тип актива — это нормально.
Но если стратегия заявлена как универсальная, стоит проверить на 2-3 парах.
Кросс-тест — это дополнительная проверка, не обязательный критерий.

### 3. Количество сделок

Чем больше сделок, тем надёжнее статистика. Ориентиры (не жёсткие правила):

| Таймфрейм | Минимум сделок за период | Почему |
|-----------|------------------------|--------|
| 1m-5m | 100+ | Много возможностей, нужна большая выборка |
| 15m-1h | 30-50 | Разумный баланс |
| 4h-1d | 15-20 | На старших ТФ сделок естественно мало |

Если сделок слишком мало — увеличь период данных или добавь пары.
Но не подгоняй стратегию чтобы она делала больше сделок — это хуже чем мало.

### 4. Форма equity curve

Посмотри график PnL в FreqUI (раздел Backtest).

Если FreqUI не показывает нужную детализацию, посмотри результаты напрямую:
```bash
# Файлы бэктеста
ls user_data/backtest_results/
# Содержат JSON со всеми сделками и метриками
```

Признаки здоровой стратегии:
- Плавный рост без резких провалов
- Нет длительных периодов "flat" (стратегия не торгует)
- Drawdown восстанавливается

Признаки проблем:
- Весь профит от 1-2 сделок
- Резкий рост в начале, потом flat
- Хаотичная кривая

### 5. Sanity checks

- **Profit vs Market change**: стратегия лучше "купи и держи"?
- **Drawdown**: ты готов пережить такую просадку реальными деньгами?
- **Avg duration**: сделки не слишком длинные/короткие для твоего стиля?
- **Win rate + Risk/Reward**: низкий win rate не значит плохая стратегия —
  если средний выигрыш значительно больше среднего проигрыша

## Подозрительные сигналы

Эти ориентиры — не абсолютные правила, а поводы задать вопрос "почему?":

| Сигнал | О чём задуматься |
|--------|-----------------|
| Profit > 200% за 3 месяца | Возможно overfitting — проверь на out-of-sample |
| Win Rate > 90% | Подозрительно стабильно — проверь логику сигналов |
| 0 убыточных сделок | Скорее всего ошибка в стратегии или данных |
| Результат резко падает при сдвиге timerange | Стратегия привязана к конкретному периоду |
| После hyperopt результат в 5× лучше | Подгонка — валидируй на отложенных данных |

Ни один из этих сигналов не значит "стратегия плохая" автоматически.
Они значат "нужна дополнительная проверка".

## Частые ошибки

1. **Оптимизировал и тестировал на одних данных** → результат нерелевантен
2. **Мало сделок для вывода** → увеличь период, не подгоняй стратегию
3. **Игнорирует drawdown** → стратегия прибыльна, но просадка неприемлема
4. **Не валидировал после hyperopt** → главный источник overfitting
5. **Ожидает одинаковых результатов на всех парах** → не все стратегии универсальны
