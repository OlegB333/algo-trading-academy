# Валидация стратегии

## Зачем

Бэктест на одной паре за один период — это только начало. Хороший результат
может быть случайностью или подгонкой. Валидация отвечает на вопрос:
"будет ли стратегия работать в реальности?"

## Чек-лист валидации

Перед тем как считать стратегию рабочей, пройди все пункты:

### 1. Out-of-sample тест

Раздели данные: 70% для разработки, 30% для проверки.

```bash
# Разработка (тренировка)
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/user_data/config/config.json \
  --strategy MyStrategy \
  --timerange 20250601-20260101

# Проверка (out-of-sample)
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/user_data/config/config.json \
  --strategy MyStrategy \
  --timerange 20260101-20260401
```

Результат на проверочном периоде не должен быть значительно хуже тренировочного.

### 2. Тест на другой паре

```bash
docker compose run --rm freqtrade backtesting \
  --config /freqtrade/user_data/config/config.json \
  --strategy MyStrategy \
  --pairs ETH/USDT \
  --timerange 20260105-20260405
```

Хорошая стратегия работает на нескольких парах, а не только на той,
на которой разрабатывалась.

### 3. Количество сделок

Минимум 30-50 сделок за период бэктеста. Меньше — статистически незначимо.
Если сделок мало — увеличь период данных или добавь пары.

### 4. Форма equity curve

Посмотри график PnL в FreqUI. Признаки хорошей стратегии:
- Плавный рост без резких провалов
- Нет длительных периодов "flat" (стратегия не торгует)
- Drawdown восстанавливается

Признаки проблем:
- Весь профит от 1-2 сделок
- Резкий рост в начале, потом flat
- Хаотичная кривая

### 5. Sanity checks

- **Profit vs Market change**: стратегия лучше "купи и держи"?
- **Drawdown**: ты готов пережить такую просадку реальными деньгами?
- **Avg duration**: сделки не слишком длинные/короткие для твоего стиля?
- **Win rate + Risk/Reward**: если win rate 30%, то средний выигрыш должен быть
  значительно больше среднего проигрыша

## Красные флаги

| Сигнал | Что значит |
|--------|-----------|
| Profit > 200% за 3 месяца | Скорее всего overfitting |
| Win Rate > 90% | Подозрительно — проверь логику |
| 0 убыточных сделок | Ошибка в стратегии или данных |
| Работает только на BTC/USDT | Подогнана под одну пару |
| Результат сильно падает при сдвиге timerange на месяц | Нестабильна |

## Частые ошибки

1. **Оптимизировал и тестировал на одних данных** → результат нерелевантен
2. **Слишком мало сделок** → статистическая случайность
3. **Игнорирует drawdown** → стратегия прибыльна, но просадка 50% — не торгуемо
4. **"Работает на BTC — значит работает"** → протестируй на 3+ парах
