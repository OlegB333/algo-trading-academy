# Версионирование стратегий

## Зачем

Каждое изменение стратегии — это эксперимент. Без версионирования через 10 итераций
невозможно вспомнить какая версия давала лучший результат. Git решает эту проблему.

## Сохранение версии

После успешного бэктеста или значимого изменения:

```bash
cd algo-trading-academy
git add user_data/strategies/MyStrategy.py
git commit -m "MyStrategy: добавил RSI-фильтр, Sharpe 1.2 → 1.8"
```

Правило: в сообщении коммита указывай **что изменил** и **как изменился результат**.

## Сравнение версий

### Посмотреть что изменилось
```bash
git diff user_data/strategies/MyStrategy.py
```

### Сравнить с предыдущей версией
```bash
git diff HEAD~1 -- user_data/strategies/MyStrategy.py
```

### Посмотреть историю изменений стратегии
```bash
git log --oneline -- user_data/strategies/MyStrategy.py
```

## Откат к предыдущей версии

### Откатить один файл
```bash
git checkout HEAD~1 -- user_data/strategies/MyStrategy.py
```

### Посмотреть конкретную версию
```bash
git show HEAD~2:user_data/strategies/MyStrategy.py
```

## Рабочий цикл

```
1. Изменил стратегию
2. Запустил бэктест (skill backtest)
3. Записал результат в коммит
4. Повторил
```

Через 10 итераций можно просмотреть `git log` и найти лучшую версию.

## Ветки для экспериментов

Для рискованных экспериментов — создай ветку:

```bash
git checkout -b experiment/macd-filter
# ... изменения, бэктесты ...
# Если результат хороший:
git checkout main
git merge experiment/macd-filter
# Если плохой:
git checkout main
git branch -d experiment/macd-filter
```

## Что коммитить

| Файл | Коммитить? |
|------|-----------|
| `user_data/strategies/*.py` | ✅ Всегда |
| `user_data/config/config.override.json` | ❌ В .gitignore |
| `user_data/backtest_results/` | ❌ В .gitignore |
| `user_data/data/` | ⚠️ Только стартовый набор |

## Частые ошибки

1. **Забыл закоммитить перед изменением** → потерял рабочую версию
2. **Коммит без описания результата** → через неделю не вспомнить что было лучше
3. **Коммит всего подряд** → git add только стратегии, не данные
