# Первый запуск проекта

## Правила

- Не исследуй файлы проекта — структура тебе известна из rules
- При нормальном старте (happy path) не показывай логи, конфиги и папки. Логи показывай **только при ошибке**
- Ученик — не программист. Отвечай кратко и по делу
- **Состояние проекта хранится в файле `.project-context`**. Все изменения инфраструктуры делай строго через `docker-compose.override.yml`, основной compose не трогать.

## Алгоритм

### 1. Определи ОС

Определи операционную систему ученика системными утилитами.
Создай файл `.project-context` в корне проекта (если его ещё нет) и запиши **реально определённую** ОС:
```env
OS=<результат определения: macOS, Windows, Linux>
```
Не копируй пример слепо — запиши именно то, что определил.

### 2. Проверь Docker

Выполни команду:
```terminal
docker compose version
```
Если команда не работает, попробуй старый синтаксис: `docker-compose version`.
Если не работают обе — попроси ученика запустить Docker Desktop.

### 3. Проверь занятость порта и допиши контекст

Проверь, свободен ли порт 8080.
**Если порт СВОБОДЕН** — допиши в `.project-context`:
```env
PORT=8080
```
**Если порт ЗАНЯТ:**
- Никогда не пытайся "убить" занятый порт или чужие процессы!
- Просто создай `docker-compose.override.yml` и смени проброс порта (например, на `8081:8080`). Основной compose.yml не меняй.
- Допиши в `.project-context` фактический порт: `PORT=8081`.

### 4. Запусти контейнер

```terminal
docker compose up -d
```
(или через дефис `docker-compose`, если сработал старый синтаксис).

### 5. Дождись healthy

```terminal
docker compose ps
```
Жди статуса `healthy`. **Сделай максимум 3 попытки.**
Если контейнер упал или так и не стал `healthy` — покажи короткий лог (`docker compose logs --tail=20`) для диагностики.

### 6. Ответь ученику

Кратко:
> ✅ Проект успешно запущен и работает.
> Веб-интерфейс FreqUI доступен по адресу http://localhost:PORT (укажи порт из `.project-context`).
