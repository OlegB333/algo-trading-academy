# Первый запуск проекта

## Правила

- Не исследуй файлы проекта — структура известна из rules
- Логи показывай **только при ошибке**
- Ученик — не программист. Отвечай кратко
- Изменения инфраструктуры — только через `docker-compose.override.yml`

## Алгоритм

### 1. Прочитай или создай `.project-context`

Если файл уже есть — прочитай `OS` и `PORT`, не перезаписывай их.

Если файла нет — определи ОС (по окружению или спроси ученика) и создай:
```
OS=macOS
```
Значение должно быть реальным, не копируй пример. PORT пока не пиши — он определится в шаге 3.

### 2. Проверь Docker

```
docker compose version
```
Если не сработало — попробуй `docker-compose version`.
Если оба не работают — попроси запустить Docker Desktop.

### 3. Проверь порт

Если `PORT` уже задан в `.project-context` — пропусти этот шаг.

Если нет — проверь свободен ли порт 8080. Если проверить не удалось — пропусти, ошибка проявится при старте.

Если порт занят — не убивай процессы. Создай `docker-compose.override.yml`:
```yaml
services:
  freqtrade:
    ports:
      - "127.0.0.1:8081:8080"
```
Обнови `PORT=8081` в `.project-context`.

### 4. Запусти и дождись healthy

```
docker compose up -d
docker compose ps
```
Жди статуса `healthy`, максимум 3 попытки.
Если не поднялся — покажи `docker compose logs --tail=20`.

### 5. Ответь ученику

> ✅ Проект запущен.
> FreqUI: http://localhost:8080

Подставь реальный порт из `.project-context`.
