# Первый запуск проекта

## Правила

- Не исследуй файлы проекта — структура тебе известна из rules
- При нормальном старте (happy path) не показывай логи, конфиги и папки. Логи показывай **только при ошибке**
- Ученик — не программист. Отвечай кратко и по делу

## Алгоритм

### 1. Проверь Docker и Compose плагин

```bash
docker compose version
```

Если команда вернула версию — Docker работает, а плагин compose установлен.
Если ошибка (например, "daemon not running" или "compose is not a docker command") — попроси ученика запустить Docker Desktop или обновить Docker.

### 2. Проверь порт 8080

Перед запуском проверь, свободен ли порт (на macOS/Linux), чтобы избежать крэша контейнера из-за Bind error:
```bash
nc -z localhost 8080 && echo "BUSY" || echo "FREE"
```
Если порт `BUSY`, попроси пользователя освободить его.

### 3. Запусти контейнер

```bash
docker compose up -d
```

### 4. Дождись healthy

```bash
docker compose ps
```

Жди статуса `healthy`. Если `health: starting` — подожди 5-10 секунд и проверь снова.
**Важно: делай максимум 3 попытки**, чтобы не зависнуть в бесконечном цикле. Если контейнер упал или так и не стал healthy — выведи логи `docker compose logs --tail=20` и помоги разобрать ошибку.

### 5. Ответь ученику

Именно так, кратко:

> ✅ Проект запущен!
>
> - **Режим:** dry-run (виртуальные деньги, ничего реального)
> - **Стратегия:** SampleStrategy (учебная, на основе SMA)
> - **Пары:** BTC/USDT, ETH/USDT
> - **FreqUI:** http://localhost:8080 (логин: `freqtrader` / `freqtrader`)
>
> Бот уже торгует виртуально. Откройте FreqUI чтобы увидеть сделки.

## Если не стартует

| Проблема | Решение |
|----------|---------|
| Docker не установлен | Ссылка: https://docs.docker.com/desktop/ |
| Docker не запущен | Запустите Docker Desktop, подождите 30 сек |
| Порт 8080 занят | `docker compose down`, закройте другой сервис, `docker compose up -d` |
| `image not found` | `docker compose pull` и снова `up -d` |
