# Первый запуск проекта

## Правила

- Не исследуй файлы проекта — структура тебе известна из rules
- При нормальном старте (happy path) не показывай логи, конфиги и папки. Логи показывай **только при ошибке**
- Ученик — не программист. Отвечай кратко и по делу

## Алгоритм

### 1. Определи и зафиксируй ОС

Выполни команду для определения операционной системы (`uname -a` или проверку среды).
Запиши эту информацию в файл `.cursorrules` в корне проекта (создай его, если нет). Формат: `Context: OS is macOS` (или Linux/WSL). Это позволит тебе сохранять контекст среды во всех диалогах.

### 2. Проверь Docker и Compose плагин

```bash
docker compose version
```

Если команда вернула версию — Docker работает, а плагин compose установлен.
Если ошибка (например, "daemon not running" или "compose is not a docker command") — попроси ученика запустить Docker Desktop или обновить Docker.

### 3. Проверь занятость порта 8080

Перед запуском проверь, свободен ли порт:
```bash
nc -z localhost 8080 && echo "BUSY" || echo "FREE"
```
**Если порт `BUSY`, реши проблему сам:**
- Сначала проверь `docker ps`. Если занят старым контейнером Freqtrade — просто сделай `docker compose down`.
- Если занят другим процессом ОС — создай `docker-compose.override.yml`, пробросив FreqUI на порт `8081:8080`.
- Если сменил порт, обязательно запиши это в `.cursorrules` (`Context: FreqUI is on port 8081`), чтобы всегда помнить.
Не заставляй ученика искать и убивать процессы.

### 4. Запусти контейнер

```bash
docker compose up -d
```

### 5. Дождись healthy

```bash
docker compose ps
```

Жди статуса `healthy`. Если `health: starting` — подожди 5-10 секунд и проверь снова.
**Важно: делай максимум 3 попытки**.

**Самостоятельное решение проблем (без участия ученика):**
- Если ошибка `image not found` — сам сделай `docker compose pull` и повтори запуск.
- При других ошибках (если не `healthy`) — сделай `docker compose logs --tail=20` для диагностики.

### 6. Ответь ученику

Когда всё `healthy`, отвечай предельно кратко (вся диагностика выше — это твоя внутренняя работа):

> ✅ Проект успешно запущен и работает.
> Включен режим симуляции (dry-run). Веб-интерфейс FreqUI доступен по адресу http://localhost:8080.
