# Первый запуск проекта

## Правила

- Не исследуй файлы проекта — структура тебе известна из rules
- Не показывай логи, содержимое конфигов, не перечисляй папки
- Ученик — не программист. Отвечай кратко и по делу

## Алгоритм

### 1. Проверь Docker

```bash
docker info --format '{{.ServerVersion}}' 2>&1 | head -1
```

Если ошибка — попроси ученика запустить Docker Desktop и подождать 30 секунд.

### 2. Запусти контейнер

```bash
docker compose up -d
```

### 3. Дождись healthy

```bash
docker compose ps
```

Жди статуса `healthy`. Если `health: starting` — подожди 10 секунд и проверь снова.

### 4. Ответь ученику

Именно так, кратко:

> ✅ Проект запущен!
>
> - **Режим:** dry-run (виртуальные деньги, ничего реального)
> - **Стратегия:** SampleStrategy (учебная, на основе SMA)
> - **Пары:** BTC/USDT, ETH/USDT
> - **FreqUI:** http://localhost:8080 (логин: `freqtrader` / `freqtrader`)
>
> Бот уже торгует виртуально. Откройте FreqUI чтобы увидеть сделки.

## Если не стартует

| Проблема | Решение |
|----------|---------|
| Docker не установлен | Ссылка: https://docs.docker.com/desktop/ |
| Docker не запущен | Запустите Docker Desktop, подождите 30 сек |
| Порт 8080 занят | `docker compose down`, закройте другой сервис, `docker compose up -d` |
| `image not found` | `docker compose pull` и снова `up -d` |
