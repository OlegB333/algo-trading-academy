# Первый запуск проекта

## Правила

- Не исследуй файлы проекта — структура тебе известна из rules
- При нормальном старте (happy path) не показывай логи, конфиги и папки. Логи показывай **только при ошибке**
- Ученик — не программист. Отвечай кратко и по делу

## Алгоритм

### 1. Определи и зафиксируй ОС

Самостоятельно определи операционную систему ученика, выполнив подходящую команду в терминале (например, `uname -a` для Mac/Linux или `ver` / `$PSVersionTable` для Windows).
Запиши результат в системный файл правил агента (например, `.cursorrules` или аналогичный автозагружаемый ИИ-контекст вашей IDE). 
Формат записи: 
`OS: [Название ОС]. Важно: Все дальнейшие консольные команды, скрипты и пути к файлам должны строго соответствовать этой операционной системе!`
Это необходимо, чтобы навсегда научить агента писать кроссплатформенные команды для компьютера ученика.

### 2. Проверь Docker и Compose плагин

```terminal
docker compose version
```

Если команда вернула версию — Docker работает, а плагин compose установлен.
Если ошибка (например, "daemon not running" или "compose is not a docker command") — попроси ученика запустить Docker Desktop или обновить Docker.

### 3. Проверь занятость порта 8080

Перед запуском проверь, свободен ли порт 8080, используя системную утилиту, подходящую для ОС ученика (например, `lsof -i :8080` на Mac/Linux или `netstat -ano | findstr 8080` на Windows PowerShell/CMD).

**Если порт ЗАНЯТ, реши проблему сам:**
- Сначала проверь `docker ps`. Если занят старым контейнером Freqtrade — просто останови его: `docker compose down`.
- Если занят другим системным процессом ОС — создай `docker-compose.override.yml`, пробросив FreqUI на альтернативный свободный порт (например, `8081:8080`).
- Если ты сменил порт, обязательно зафиксируй это в файле правил агента вместе с версией ОС (`Важно: FreqUI запущен на порту 8081`), чтобы всегда помнить это в будущих задачах.
Не заставляй ученика искать и убивать процессы.

### 4. Запусти контейнер

```terminal
docker compose up -d
```

### 5. Дождись healthy

```terminal
docker compose ps
```

Жди статуса `healthy`. Если `health: starting` — подожди 5-10 секунд и проверь снова.
**Важно: делай максимум 3 попытки**.

**Самостоятельное решение проблем (без участия ученика):**
- Если ошибка `image not found` — сам сделай `docker compose pull` и повтори запуск.
- При других ошибках (если не `healthy`) — сделай `docker compose logs --tail=20` для диагностики.

### 6. Ответь ученику

Когда всё `healthy`, отвечай предельно кратко (вся диагностика выше — это твоя внутренняя работа):

> ✅ Проект успешно запущен и работает.
> Включен режим симуляции (dry-run). Веб-интерфейс FreqUI доступен по адресу http://localhost:8080.
