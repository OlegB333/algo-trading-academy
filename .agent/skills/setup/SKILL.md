# Первый запуск проекта

## Правила

- Не исследуй файлы проекта — структура тебе известна из rules
- При нормальном старте (happy path) не показывай логи, конфиги и папки. Логи показывай **только при ошибке**
- Ученик — не программист. Отвечай кратко и по делу
- **Состояние проекта хранится в файле `.project-context`**. Все изменения инфраструктуры делай строго через `docker-compose.override.yml`, основной `docker-compose.yml` не трогать.

## Алгоритм

### 1. Определи ОС

Если файл `.project-context` уже существует и в нём задана `OS=` — **не перезаписывай**, используй то что есть.

Если файла нет или `OS` не задана — определи ОС по окружению или доступной команде. Если определить не удалось — спроси у ученика.
Создай файл `.project-context` в корне проекта и запиши **реально определённую** ОС:
```
OS=macOS
```
(вместо `macOS` — фактическое значение: `macOS`, `Windows`, `Linux`).

### 2. Проверь Docker

Выполни:
```
docker compose version
```
Если не сработало — попробуй старый синтаксис:
```
docker-compose version
```
**Запомни, какой синтаксис сработал** — сохрани в `.project-context` и используй его во всех дальнейших командах.

Если не работают оба — попроси ученика запустить Docker Desktop.

### 3. Проверь занятость порта и допиши контекст

Попробуй проверить, свободен ли порт 8080 (способ зависит от ОС).
Если проверить не удалось — не блокируй запуск, просто переходи к шагу 4 (если занят, то ошибка порта проявится при старте контейнера).

**Если порт СВОБОДЕН** — допиши в `.project-context`:
```
PORT=8080
```

**Если порт ЗАНЯТ:**
- Никогда не пытайся "убить" занятый порт или чужие процессы!
- Создай файл `docker-compose.override.yml` в корне проекта и смени проброс порта (например, `8081:8080`). Основной `docker-compose.yml` не меняй.
- Допиши в `.project-context` фактический порт: `PORT=8081`.

### 4. Запусти контейнер

Используй тот синтаксис Docker Compose, который сработал в шаге 2:
```
docker compose up -d
```

### 5. Дождись healthy

```
docker compose ps
```
Жди статуса `healthy`. **Сделай максимум 3 попытки.**
Если контейнер упал или так и не стал `healthy` — покажи короткий лог (`docker compose logs --tail=20`) для диагностики.

### 6. Ответь ученику

Кратко, подставив реальный порт из `.project-context`:

> ✅ Проект успешно запущен и работает.
> Веб-интерфейс FreqUI доступен по адресу http://localhost:8080

(если порт был сменён — указать фактический, например `http://localhost:8081`).
