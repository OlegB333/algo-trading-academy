# Telegram-интеграция

## Зачем

Telegram позволяет:
- Получать уведомления о каждой сделке (вход, выход, прибыль)
- Видеть статус бота (работает, остановлен, ошибки)
- Управлять ботом удалённо (старт, стоп, статистика)
- Получать дневные/недельные отчёты

## Безопасность

Токен и chat_id — это секреты. Правила:
- Хранить **только** в `.env`
- **Не коммитить** в git (`.env` уже в `.gitignore`)
- **Не показывать** в чате
- **Не помещать** в `config.override.json` или другие файлы в репозитории

## Шаг 1: Создание Telegram-бота

1. Открой @BotFather в Telegram
2. Отправь `/newbot`
3. Придумай имя и username
4. Скопируй **токен** (формат: `123456:ABC-DEF...`)

## Шаг 2: Получение chat_id

1. Отправь любое сообщение своему новому боту
2. Открой в браузере:
   ```
   https://api.telegram.org/bot<ТОКЕН>/getUpdates
   ```
3. Найди `"chat":{"id":123456789}` — это твой chat_id

> Этот способ передаёт токен в URL браузера. Используй только для первоначальной
> настройки, после получения chat_id закрой вкладку и очисти историю браузера.

## Шаг 3: Сохранение в .env

```bash
# В файле .env (на VPS)
TELEGRAM_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
TELEGRAM_CHAT_ID=123456789
```

## Шаг 4: Настройка Freqtrade

Telegram-конфигурация передаётся через переменные окружения или конфиг.
Так как токен — секрет, настройка Telegram происходит на этапе VPS-деплоя
совместно с агентом, который поможет безопасно интегрировать `.env` переменные
в конфигурацию бота.

## Уведомления (пассивные)

После подключения бот автоматически присылает:

| Событие | Что приходит |
|---------|-------------|
| Старт бота | Приветственное сообщение |
| Вход в позицию | Пара, цена, размер |
| Выход из позиции | Пара, прибыль/убыток, причина выхода |
| Ошибка | Описание проблемы |

Настройка какие уведомления включить/выключить:
```json
{
    "telegram": {
        "notification_settings": {
            "status": "silent",
            "warning": "on",
            "startup": "on",
            "entry": "on",
            "exit": "on"
        }
    }
}
```

## Команды управления (удалённый контроль)

Помимо уведомлений, Telegram позволяет **управлять ботом удалённо**.
Это мощная функция — относись к ней с осторожностью.

### Информационные (безопасные)

| Команда | Что делает |
|---------|-----------|
| `/status` | Текущие открытые позиции |
| `/profit` | Прибыль за сегодня/неделю/месяц |
| `/balance` | Баланс на бирже |
| `/daily` | Дневная статистика |
| `/weekly` | Недельная статистика |
| `/performance` | Результаты по парам |
| `/help` | Список всех команд |

### Управляющие (влияют на торговлю)

| Команда | Что делает | Риск |
|---------|-----------|------|
| `/start` | Возобновить торговлю | Бот начнёт входить в сделки |
| `/stop` | Остановить торговлю | Позиции остаются открытыми |
| `/stopentry` | Запретить новые входы | Существующие позиции не закрываются |
| `/forcesell <id>` | Принудительно закрыть позицию | Закроет по текущей цене, возможно с убытком |

Управляющие команды — это по сути удалённый доступ к торговому счёту.
Убедись что Telegram-аккаунт защищён двухфакторной аутентификацией.

## Проверка работы

1. Запусти бота с Telegram-конфигурацией
2. Бот пришлёт приветственное сообщение при старте
3. Отправь `/status` — должен ответить

Если бот не отвечает:
- Проверь токен и chat_id
- Убедись что отправил сообщение боту ДО запуска (нужен getUpdates)
- Проверь логи: `docker compose logs -f --tail 20 | grep -i telegram`

## Частые ошибки

1. **Не отправил сообщение боту до запуска** → chat_id не определится
2. **Токен в коммите** → скомпрометирован. Перегенерируй через @BotFather
3. **Токен в override-файле** → может попасть в git. Только в `.env`
4. **Нет 2FA на Telegram** → при взломе аккаунта злоумышленник управляет ботом
5. **Telegram заблокирован провайдером** → нужен VPN или proxy в config
