# Telegram-интеграция

## Зачем

Telegram позволяет:
- Получать уведомления о каждой сделке (вход, выход, прибыль)
- Видеть статус бота (работает, остановлен, ошибки)
- Управлять ботом командами (старт, стоп, статистика)
- Получать дневные/недельные отчёты

## Шаг 1: Создание Telegram-бота

1. Открой @BotFather в Telegram
2. Отправь `/newbot`
3. Придумай имя и username
4. Скопируй **токен** (формат: `123456:ABC-DEF...`)

## Шаг 2: Получение chat_id

1. Отправь любое сообщение своему новому боту
2. Открой в браузере: `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. Найди `"chat":{"id":123456789}` — это твой chat_id

## Шаг 3: Настройка

Токен и chat_id — это секреты. Храни их в `.env`:

```bash
# В файле .env
TELEGRAM_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
TELEGRAM_CHAT_ID=123456789
```

Создай `config.override.json`:
```json
{
    "telegram": {
        "enabled": true,
        "token": "YOUR_TOKEN_HERE",
        "chat_id": "YOUR_CHAT_ID_HERE"
    }
}
```

> ⚠️ На реальном VPS токен лучше передавать через переменные окружения,
> а не хардкодить. Для учебного dry-run — допустимо в override.

## Команды управления через Telegram

После подключения бот отвечает на команды:

| Команда | Что делает |
|---------|-----------|
| `/status` | Текущие открытые позиции |
| `/profit` | Прибыль за сегодня/неделю/месяц |
| `/balance` | Баланс на бирже |
| `/daily` | Дневная статистика |
| `/weekly` | Недельная статистика |
| `/performance` | Результаты по парам |
| `/start` | Возобновить торговлю |
| `/stop` | Остановить торговлю (позиции остаются) |
| `/stopentry` | Запретить новые входы |
| `/forcesell <trade_id>` | Принудительно закрыть позицию |
| `/help` | Список всех команд |

## Настройка уведомлений

В override можно тонко настроить что присылать:

```json
{
    "telegram": {
        "enabled": true,
        "token": "YOUR_TOKEN",
        "chat_id": "YOUR_CHAT_ID",
        "notification_settings": {
            "status": "silent",
            "warning": "on",
            "startup": "on",
            "entry": "on",
            "exit": "on"
        }
    }
}
```

## Проверка работы

1. Запусти бота с override, включающим Telegram
2. Бот пришлёт приветственное сообщение при старте
3. Отправь `/status` — должен ответить

Если бот не отвечает:
- Проверь токен и chat_id
- Убедись что отправил сообщение боту ДО запуска (нужен `getUpdates`)
- Проверь логи: `docker compose logs -f --tail 20 | grep -i telegram`

## Частые ошибки

1. **Не отправил сообщение боту до запуска** → chat_id не определится
2. **Токен в коммите** → скомпрометирован. Перегенерируй через @BotFather
3. **Несколько аккаунтов в Telegram** → chat_id пишется в тот аккаунт, из которого писал боту
4. **Telegram заблокирован провайдером** → нужен VPN или proxy в config
